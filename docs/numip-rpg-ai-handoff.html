  5. 專案目前有未提交修改。先讀實際檔案與 git diff， 不要假設 README 是最新規格。
  6. 題面不能靠「加一段場景前綴」或大量關鍵字替換假裝融入劇情； 必須回答「為何角色現在需要算這題，答案如何推動場景」。

1. 一頁摘要

  • 產品：Numip RPG，數感宇宙 IP 下的純文字數學冒險；原下注、錢包、連勝、破產救濟等 Math Wager 機制全部移除。
  • 進入流程：輸入名字 → 選年段 → 選數學家劇本 → 玩家本人與 NumNum、Numi 一起冒險。
  • 內容矩陣：5 位數學家 × 6 個國中年段 = 30 本；目前只做第一本「七上 × 阿基米德」。
  • 單局：固定史實主線與結局，穿插 15 個抽題錨點；題目從對應年段題庫抽取，同局不重複。
  • 答錯：進入短繞路並抽同知識點、相近難度題； 最多 2 道類似題，之後強制接回主線。
  • 正確率：答對題數 ÷ 實際作答題數，主線題與繞路題全部計入。
  • 介面：全螢幕、溫暖藍紫終端機感、一次一句打字機； Enter 可補完／推進，保留最近 3 句並淡出舊句。
  • 資料：1156 題本機題庫；劇本 JSON 資料驅動；本機 localStorage 單一存檔。

2. 產品定位與非目標

產品定位

讓國中數學題不再像與故事無關的插題，而是成為歷史場景中的實際行動。 玩家不是旁觀者，也不是固定姐弟角色；玩家以自己輸入的名字進入劇本， 與 NumNum、Numi 共同經歷數學家的關鍵人生片段。

目前非目標

  • 不保留任何下注、錢包、倍率或 streak 玩法。
  • MVP 不登入、不建立帳號、不做雲端存檔。
  • 不讓 AI 即興生成數學題或答案；題目只能來自既有題庫。
  • 不做可改變史實終點的多結局。
  • 尚未開始產出其餘 29 本劇本。

3. 數感宇宙 IP：不可擅改的核心

世界觀

  • 現實不存在完美直線、完美幾何與完美長度；完美只存在平行的「數學世界」。
  • 數學世界乾淨、藍紫、極安靜；人間場景使用暖色形成對比。
  • 人類經過漫長思考真正「想通」時，會爆發暖黃色能量 Eureka，能穿越時空召喚數學世界的存在。
  • Eureka 對 NumNum、Numi 像甜點與能量食物。

角色

角色 固定設定
玩家 使用者輸入名字後本人進入劇本，不是旁觀者。
NumNum 愛思考與秩序，常整理數字；謹慎，語氣簡短。
Numi 行動派、衝動、愛追黃金螺旋，常撞亂 NumNum 的數列。
πmi NumNum 在數學世界打造的思考機器。屬於起源故事，但目前第一本冒險的固定同伴仍是 NumNum + Numi，不是 πmi。
歷史人物 阿基米德、伽利略、牛頓、高斯、艾倫・圖靈。

Eureka 規則

  • 只有使用者或數學家真正理解／學會某個知識的瞬間才出現。
  • 進入劇本不靠 Eureka，而是被遠方微光與細聲吸引。
  • 一般答對只有場景文字回饋，不閃光、不喊 Eureka。
  • 第一本文案只在大節點觸發：浴缸浮力、槓桿原理；若節點綁題， 必須答對／通過後才觸發。

5. 玩家流程與狀態機

title → name（輸入玩家名） → grade（七上／七下／八上／八下／九上／九下） → script（選數學家） → resume（若該劇本有存檔） → play → ending（顯示正確率）

主線節點

narrative / eureka → 打字機逐句播放 → Enter 推進 → next 節點 quiz → 播放 setup 台詞 → 抽題並做場景改寫 → 顯示題幹與選項／數字輸入 → 正確：場景回饋 → onCorrect → 錯誤：繞路台詞 → 同知識點相近難度題 → 類似題最多 2 道 → 超過後強制接回 onCorrect

答題規則

  • 每本固定 15 個抽題錨點，不綁死單一題目。
  • 同一局排除已抽過的 questionId。
  • 類似題優先同知識點、相近難度；題型可以在選擇與填答間變化。
  • 答錯不改變結局，只增加路程。
  • 正確率 = correctCount / attemptCount，所有實際作答都計入。

6. 打字機與互動規格

  • 一次顯示一句，逐字出現；目前速度 TYPE_MS = 28。
  • 打字中按 Enter：立即補完整句。
  • 句子已完整時按 Enter：顯示下一句或進入下一節點。
  • 畫面最多保留最近 3 句；舊句依序降到 52%、28% 透明度。
  • 答題前的 setup 也走同一隊列，播完才揭露題目。
  • 選擇題點選後，焦點會抓回遊戲殼層；這是為了修復「答完後按 Enter 沒反應」。
  • 選擇題可點 A–D；填答題在輸入列輸入後按 Enter 或提交。

7. 題面劇情化：已鎖定原則

  • 整段情境要換成當下劇本場景，不只加旁白。
  • 數字、算式、未知數、選項字母與正確答案不得改變。
  • 原本是文字題也要整段重寫；選項文字同步改成劇本語彙。
  • 純算式可保留數學核心，但提問原因必須屬於當下行動。
  • 禁止「【場景】」「照板上的紀錄算」等外掛標籤；題面本身就是劇情。
  • MVP 由人工模板／精選題面負責，保留 adaptQuestionAI() 接口；AI 失敗時回原題。

10. 存檔模型

  • MVP 純 localStorage，無登入。
  • 每個劇本設計上只有一個最新存檔；目前實作實際上是全站單一 key。
  • 每過節點或答完題自動寫入。
  • 新開一局才播放「微光吸引」開場；讀檔從存下的節點重播。
  • 存下玩家名、劇本 ID、節點、已用題 ID、答對數、作答數與 quiz。

13. 建議下一步（依優先順序）

  1. 讓使用者實玩錨點 1–3： 收集題面自然度、節奏、字級與回饋意見；先不要擴大範圍。
  2. 修正前三錨點細節： 若數字雖正確但場景尺度不自然，應更換精選題或重寫情境， 不可只做文字替換。
  3. 精修錨點 4–15： 每個錨點先定義「角色目的 → 題目動作 → 答案造成的場景結果」， 再建立精選題 ID 與逐題改寫。
  4. 補測試： 先針對 questionPool、checkAnswer、detour 上限、accuracy、存檔恢復、 Enter 推進建立單元／互動測試。
  5. 重構 App： 將遊戲狀態改為 reducer／狀態機，抽出 useTypewriter、useGameSave 與 Quiz 元件。
  6. 清理舊專案： 更新 README、移除未使用依賴與資產、修 lint warning。
  7. 確認第一本後才擴充： 建立劇本 schema validator 與腳手架，再生產其餘 29 本。
  8. 最後處理命名部署： GitHub repo 改為 numip-rpg，Vercel 專案與網域同步更新。

  • 數字、算式、未知數、選項字母、答案未改。
  • 題面本身成立為當下場景，不靠標籤或前綴。
  • 同局不重複；繞路題不是剛答錯的原題。
  • 最多 2 道類似題後強制接回主線。

操作

  • 打字中 Enter 補完；打完 Enter 下一句。
  • 答完選擇題後 Enter 仍可推進。
  • 手機與桌面皆全螢幕且字級可讀。
  • 遊戲中無 header、無題數進度。
  • 中斷後可從存檔節點繼續，統計不遺失。
